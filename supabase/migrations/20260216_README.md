## Why Two Files?

PostgreSQL doesn't allow adding a new enum value and using it in the same transaction. The `ALTER TYPE ... ADD VALUE` command must be committed before the new value can be referenced in any queries or functions.

## Quick Run (Copy-Paste Approach)

If you want to run everything at once in the Supabase SQL Editor:

1. Copy the contents of `20260216_01_add_expired_enum.sql`
2. Paste and **RUN** in SQL Editor
3. Wait for it to complete
4. Copy the contents of `20260216_02_booking_request_system.sql`
5. Paste and **RUN** in SQL Editor

## Troubleshooting

**Error: "unsafe use of new value 'expired'"**
- This means you tried to run part 2 before part 1
- Solution: Run part 1 first, wait for completion, then run part 2

**Error: "type 'booking_status' already exists"**
- This is fine, the enum already exists
- Continue with running the migrations

**Error: "enum label 'expired' already exists"**
- This is fine, the value was already added
- Continue with running the migrations
